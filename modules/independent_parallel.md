# Independent Parallel
**Independent Parallel** とは、一つのテーマを複数人が別々に取り組むスタイルを指す。

従来のチームワークは一つのテーマに皆で協調して取り組むものであった。1-テーマ 1-アウトプットであり、このアウトプットの質とスピードを高めるためにチーム全員で一丸になる。

しかし Independent Parallel では、1-テーマ 1-アウトプットをひとりで行い、これが n 人分存在する。仮に 5 人チームの場合、アウトプットも 5 つである。なお、Independent Parallel の概念自体は、どのアウトプットを選ぶか、またこれらをどのように統合するかといったことは扱わない。あくまでも各人が別々に取り組む様を指す。

## Independent Parallel の主目的
チームワークによる初期段階の探索を進化させることである。

たとえ探索かつ初期段階であっても、チームワークである以上、既存のコンテキストやコアメンバーとの実力差の影響を受けてしまう。本質的な探究よりも、コンテキストの理解とコアメンバーへの迎合にエネルギーを費やす羽目になる。Independent Parallel を使うと、全員が自分の城を持ちながら、しかし他の人の取り組みから影響を受けることもできる。チームワークほど政治的ではなく、個人ほど孤独でもない、バランスの良い第三のあり方なのだ。

特に 2026 年 2 月現在では、生成 AI により一個人の能力が引き上げられている。ソフトウェア開発にせよ、概念開発や組織開発にせよ、あるいは創作にせよ、多くの分野において Independent Parallel は現実的に可能なのである。

## エクスプロラトリとの関連
Independent Parallel は、エクスプロラトリを成立させるための一条件である。
