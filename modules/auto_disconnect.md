# 時限切断
**時限切断（Auto Disconnect）** とは、指定時間経過後にイベント（特にミーティング）を強制切断することを指す。たとえば 30 分を設定した場合、そのミーティングは 30 分後に自動で全員が切断される。

すでに Zoom のブレイクアウトルームがこの機能を持っているが、時限切断はブレイクアウトルームに限らず、あらゆるオンラインミーティングに当てはめることができる。むしろ、チームの文化として採用できるものである。

時限切断を使うと、全員が残り時間を意識することになりミーティングの質が潜在的に向上する。また、一度強制切断された後で再び集まる動きはしづらいため、ミーティングが無駄に長引くのも防止できる。あるいは、それでも構わずミーティングを続ける者がいた場合、それはそういう者（区切りや休憩を意識しない無能な人間）であることをあぶり出せたと言える。総合的に見て、時限切断はミーティングの体験を向上させる。

最大のハードルは実装であり、時限切断を採用したいなら時間指定と切断動作の両方はシームレスにしなくてはならない。ファシリテーターがユーザーにお願いするといった人力の運用は NG である。時限切断は、指定時間が来たら一瞬で全員を切断しなければならない。そこまで徹底するからこそ「自動で切断される」とのメンタルモデルが醸成される。ここまで徹底できないのなら、時限切断を導入する意味はない。Zoom のように API 豊富な会議ツールか、そうでなければ時限切断可能な手段を別途用意しなくてはならない。

また時限切断に「例外的な延長」の概念はない。指定時間後に必ず切れるという世界観をシンプルに実現するため、時限切断は例外を持たない。
