# アダプタビリティ
**アダプタビリティ（Adaptability）** とは、その組織や仕事に適応する能力を指す。ソフトスキルの一種であり、暗黙のうちにほぼ必ず要求される重要スキルである。

いくらハードスキルが高くとも、その組織の文化・ルール・実態に適用できなければ戦力とはみなされない。下手をすると有害とみなされて排除されるだろう。たとえばフル出社を社員全員に化している組織に入った場合、あなたもフル出社しなければならない。これは事実上、フル出社に納得し、フル出社を継続し、またそのような現状に耐え続けるだけの能力が必要ということである。

## アダプタビリティの対象
体系化されてはいないが、たとえば以下が存在する:

- 慣例（Convention）
    - 暗黙的なルールを指す。特に社内規定そのものではなく、それを踏まえた実際の運用を指す
    - たとえば規定上は成果主義的な評価プロセスになっているが、実態としては横並びとして運用されている場合、慣例も横並びの運用である。仮に顕著な成果を出し、規定も示した上で高評価を要求したとしても、慣例上受理されることはないだろう
- 慣習（Custom）
    - 暗黙的な文化を指す。正式な文書の形で明文化されておらず、しかし事実上全員が守らねばならないレベルの圧力が働いている
    - たとえば半期ごとに期首キックオフと期末キックオフがあり、かつキックオフ後に懇親会があって、そこでの顔合わせとアピールが政治上必須となっている場合、キックオフへの参加は（評価されたいなら）必須だろう。仮にこの点を言語化し、より効果的な評価方法やアピール方法を提案したとしても、おそらく相手にされまい。文化は強行しない限りは変えられないからだ
- 慣行（Practice）
    - 現場の実態を指す
    - 変えるのは基本的に難しい。なぜなら今現在の運用により回っている仕事が存在しており、かつこれを止めることができないためである。また、今の運用に従うだけの受動的な人間が多く（そもそも主体的に改善を行える者はマイノリティである）、変えるのに必要なスキルやマインドがない。さらに今の仕事で忙しくてリソースがない
- ドメイン（Domain）
    - 業界・分野・組織固有の知識を指す。抽象的には世の中に広く知られているものもあるが、基本的にその範囲内に閉じており、関係者のみが知れるようになっている、つまりは機密性がある。
    - たとえば建築業界に携わる場合、建築に関する様々な知識が必要だろう。また業界全体で通じる慣例や慣習もドメインとみなせる。主要なプレイヤーの存在とその傾向、またパワーバランスなどは典型例だ
