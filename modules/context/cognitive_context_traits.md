# 認知的コンテキスト特性
**認知的コンテキスト特性（Cognitive Context Traits）** とは、人間の認知能力を「コンテキスト」でたとえて表現するモデルである。以下 CCT と略す。一言で言えば、口頭コミュニケーションに必要な短期記憶が乏しいことを「コンテキストウィンドウが狭い」とたとえる類のたとえである。このたとえを拡張して、4つの軸を用意したものだ。

## CCT の診断
CCT では、以下の 4 つの軸を使う。

- Window：コンテキストウィンドウの広さ
    - 脳内で処理を完結させるために必要な情報を留めておく能力
    - 短期記憶もここに含まれる
- Switching：コンテキストスイッチングの速さ
    - 異なる文脈に適応するために、ウィンドウ内の情報を入れ替える能力
- Compress：コンテキスト圧縮の速さ
    - ウィンドウに入り切らない情報を要約したり、抽象化したり、長期記憶を参照するポインタに変換したりする能力
- Archive：コンテキスト倉庫の広さ
    - ウィンドウに入り切らず、かつ圧縮しても入り切らない情報を外部化しておき、そこから素早くウィンドウに読み込むための一連の仕組み。また、それを扱ったりメンテナンスしたり強化したりする能力
    - 読み書き能力やノートテイキングの仕組みとスキルはここに含まれる

各軸に対して High、Middle、Low の 3 段階を評価する。これはチームや組織など何らかの範囲内における自分の立ち位置を示す。

- High：速い、または広い。人並以上の能力があり、周囲がしばしばついてこれない
- Middle：平均的。特に困っておらず、通常 CCT を意識することはない
- Low：遅い、または狭い。周囲よりも明らかに劣っており、コミュニケーション全般に支障が生じる。

評価は自己評価でもいいし、範囲内の他者に尋ねたり、皆で議論しても良い。録画データ（音声）から生成 AI を用いて評価させることも可能だ。

## 例
評価を行うと、人ごとに以下のようなテーブルができるはずだ。いくつか例を示す。

例1: 以下はウィンドウが狭く、スイッチングも遅いが、倉庫が広いという特性を示している。おそらくすべてのミーティングとタスクにおいてメモを取っているだろう。

|           | Low | Middle | High |
| --------- | --- | --- | --- |
| Window    | o |  |  |
| Switching | o |  |  |
| Compres   |  | o |  |
| Archive   |  |  | o |

例2: 以下は多くの会社で最も多いタイプであり、口頭コミュニケーションを支えるだけのウィンドウの広さと、慌ただしくなりがちな仕事と人間関係に対処できるマルチタスクの能力（つまりはスイッチングの速さ）を持っているが、反面、圧縮と倉庫を鍛える機会がなく、貧弱である。それはつまり抽象と具体を行き来したり、概念を用いた議論をしたり、あるいは読み書きに基づいた深い思考や非同期的なコミュニケーションが行えないことと同義であり、ミーティングと政治がほぼ必然的に生じることとなる。
