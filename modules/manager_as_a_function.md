# Manager As A Function
**機能としてのマネージャー(Manager As A Function)** とはマネージャーを機能とみなし、メンバーが必要に応じてこれを使うという考え方。マネージャーは自身の部下またはメンバー n 人に対して、機能としての自身を提供する。

## 提供方法の例
- Direct Appointment
    - マネージャーの予定表の空いているところに、誰でも自由に予定を追加していいとする
- AMA(Ask Me Anything)
- RMA(Request Me Anything
- オフィスアワー
- マネージャー・チケット
    - チケットとは「マネージャーに 1 時間分の検討と回答をお願いできる」もの
    - マネージャーはチケットをメンバーに発行し、使ってもらう
    - チケットによる作業は非同期的に行い、回答はメッセージやドキュメントの形で行う。プロトタイプなど実装をしても良いが必須ではない

MaaF は同期的な提供を想像しがちだが、非同期的な提供も可能である。重要なのは使う側のニーズに合わせることである。おそらく同期を好む者、非同期を好む者、同期でないと伝えづらいこと、非同期の方が伝えやすいことなどがあるため両方用意しなければならない。

## マネージャーがメンバーに提示する「モード」の例
- MaaF モード
    - MaaF として動く
- Solo モード
    - 個人ワークを行う
    - MaaF もしないし、ミーティングもしない
- Default モード
    - MaaF でも Solo でもない通常の状態
    - マネージャーとしておそらくミーティングに従事している
    - MaaF はしないか、滞留しているものを消化する程度

モードの概念を導入し、また表示することによって、使う側も状況の把握や予測がしやすくなる。
