# デイリーエリア
**デイリーエリア** とは、日単位で新たに作成する共用ワークスペースを指す。

以下をすべて満たさなくてはならない:

- リアルタイムな同時編集が可能な手段を使うこと
- エディット領域であること
- 誰の書き込みであるかを行レベルで識別できること or そのための記法を導入すること
- 日単位で場所を設け、かつ場所の名前は 2026/02/02 や 2026-03-02 など日を示すこと

QWINCS、特にウィキやノートやボードであれば満たせるだろう。

この前提で、各自が自分の記入領域を確保し、そこに日々の日記や備忘録やログを書いていく。したがってメンバーが 5 人いる場合、デイリーエリアには 5 つのスペースが確保されるはずだ。

また書き込みは行指向的またはブロック指向的に行う。これにより「1つの話題を1つの行やブロックに閉じ込める」ことで、情報の再利用性とコメントのしやすさを確保できる。

## Markdown によるイメージ
Markdown ファイルでは上記要件を満たせないだろうが、イメージを挙げておく。

```
# 2026-02-02

全体連絡:

- ...
- ...

A:

- ...
- ...

B:

- ...
- ...

C:

- ...
- ...
```

上記は 2026-02-02 のデイリーエリアであり、2026-02-02 の間はメンバー全員、ここに出入りすることになる。各個人の領域は「メンバー名 + リスト」の形で確保されている。

また「全体連絡」などメンバー以外のカテゴリーも導入しているが、いたずらにカテゴリーを増やしても混乱するため、なるべく増やすべきではない。多くとも 2 個、できれば 0 個に留めたい。メンバー以外のカテゴリーが恒常的に発生する場合は、問題が生じていると考えるべきである。デイリーエリアはメンバー全員が高頻度にアクセスし、内容も把握しているものだからである。おそらく自発的に読み書きしない怠惰な者がいるか、人数が多すぎる（一桁が良い）か、全体で抱えるタスクが多いか、マネジメントがマイクロ寄りか、のいずれかである可能性が高い。

リストは行指向的でもあるため、他の人が割り込んで書き込むこともできる（下記では B がコメントを書いてる）。

```
A:

- ...
    - B: ...
    - A: ...
        - B: ...
- ...
```

一方、下記のように、A の領域を侵食せず引用ベースで書くこともできる。

```
A:

- XXXXX
- ...

B:

- ...
- > XXXXX
    - ...
```

また、このとき話題 XXXXX が盛り上がったとするならば、おそらく別のページをつくった方が良い。2026-02-02.md ではなく XXXXX.md をつくり、2026-02-02.md からは XXXXX.md にリンクすると良い。これで XXXXX は再利用・再参照可能なページとなったし、議論が長くなって 2026-02-02 のスクロールバーを小さくすることもなくなる。
