# Casual Framework
難解な概念 XXXX をカジュアルに取り入れるために、すぐに使える軽量な概念的道具を揃えておくことを **カジュアリング（Casualing）** と呼びたい。

Casual Framework とは、XXXX のカジュアリングを行うためのフレームワークである。

## つくりかた
- 1: XXXX を決める
- 2: Casual XXXX と名付けた後、これが何のための道具集であるかを定める
- 3: 道具を拡充していく
    - カタログに n の道具がある、という構成にする
    - また道具がどのようなものであるかも定めると良い

## 例
- Casual Innovation
    - イノベーションの発生確率を上げるため
    - 道具の定義はなし
- Casual Ajile
    - アジャイル開発の本質を実践するため
    - 道具は「プラクティス」と定義している

## 参考
道具の種類については、「セブンリスト原則」や「MPRS」が参考になるだろう。
