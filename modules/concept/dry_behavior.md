# ドライな振る舞い
**ドライな振る舞い** とは、生成 AI に対する態度のようにドライに振る舞うことを指す。

具体的には以下をすべて満たす:

- 「言語で入力を与える」「その入力を踏まえて処理をしてもらうのを待つ」「処理の出力が来るので確認し、フィードバックや続きも言語化して再び入力する」のフィードバックループを行う。相手をその対象とみなす
- 非同期的にコミュニケーションを行う

## 人間への適用
ドライな振る舞いは人間に対して適用することを意図している。無論、こんな非人間的な態度は通常ハラスメントにしかならないが、これが有効となることもある。

有効となる例:

- 一部のニューロダイバージェントなど、人間的で同期的なコミュニケーションが不得手な者
- 「機能としてのマネージャー」など、割り切ったコミュニケーションを実施したい者
- 集中を要する作業や探索など、コミュニケーションが邪魔となるような状況においてマネジメントを行いたい者

また、適用は常に行うとも限らない。モードとして扱うと良い。たとえば「ドライ・モード」をつくって、このモードの間はドライに振る舞うようにする。スケジューラーで午前中にドライ・モードを入れておけば、午前中はドライに接してくださいね、と示せる。

## サイクルの設計
ドライな振る舞いはサイクルとして設計した方が良い。

### 例1: PDCA サイクル
- Plan：相手に渡す文面を考える
- Do：相手に文面を送信する
- Check：相手の返事が来たら、確認する
- Act：次の文面を考えて渡すか、いったん終わる

しかし PDCA は抽象的すぎてわかりづらいかもしれない。

### 例2: 4Pサイクル
4P は以下から成る:

- 1: Prompt。相手に渡す文面をつくって渡す
- 2: Polling。相手からの返事を待つ。ポーリングすれば良い
- 3: Proofread。返事が来たら対処するが、こちらで校正するつもりで望む
    - つまり相手は下書きをつくる段階まで行うのであり、以降の清書はこちらでやるとの分担を敷く
- 4: Plan。引き続きやりとりするのか、それともいったん終えるのかを考える

```
  +---> 1:Prompt ----+
  |                  |
  |                  V
4:Plan           2:Polling
  A                  |
  |                  |
  +-- 3:proofread <--+
```

4P サイクルは、ドライな振る舞い用に一から開発したサイクルであり、P- 始まりで、かつ各段階も具体的な指示に基づいているのが特徴である。PDCA サイクルよりも具体的で使いやすい分、汎用的ではない可能性がある。
