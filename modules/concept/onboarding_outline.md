# オンボーディング・アウトライン
**オンボーディング・アウトライン** とは、オンボーディングコンテンツをアウトライナー的な箇条書きで目次化したものである。

## フォーマットと例
markdown における例を挙げる。

```markdown
- [text](link)
- text
===
- text
- text
    - [text](link)
    - text
- text [text](link)
===
- text
    - [text](link)
        - ★レベル2以降は禁止
- [text](link) ★これは許されない。サブリストをぶら下げているのでリンクは付けてはならない
    - [text](link)
    - [text](link)

```

## 原則
- 1コンテンツ1行で書き、一行に収まらない内容は必ず別ページに逃がしてそこにリンクする
    - コンテンツとは「きりのいい作業や準備の単位」を指す
- コンテンツは上から順に、実施する順に並べる。また順序を問わないものは `===` で区切られた部分で表現する
- レベル 2 以上のサブリストをつくってはならない
- レベル 1 のサブリストをつくる場合、レベル 0 のコンテンツは「ラベル」となる。ラベルとは下位レベルのコンテンツに対するタイトルであり、それ自体はコンテンツを持たない。したがってリンクもしないはずである

## 設計として
オンボーディング・アウトラインは、オンボーディングの設計書としても使える。つまり、まずはリンクを一切つくらずに書く。そうして全体構造を設計した後、各コンテンツを拡充していけば良い。

## 連携の仕方
- 同時編集可能なノートアプリでオンボーディング・アウトラインをつくるなら、複数人で同時編集しながら仕上げていけるだろう。
- 単に各メンバーにつくらせて、それを集めた上で代表者が整理することもできる。

このように、オンボーディング・アウトラインは連携もしやすい。
