# Collaboration Modules

## このリポジトリの概要
このリポジトリは、コラボレーション（複数人による協業）を柔軟に行う際に使える「概念的な道具」を取り揃えている。これを **コラボレーションモジュール** と呼ぶ。このリポジトリは、筆者 [sta](https://github.com/stakiran) が開発したコラボレーションモジュールを取り揃えている。

例1: ウォーターフォール（厳密なタスクとスケジュールに基づいた計画とその管理）的な仕事の仕方しか知らない組織が、生成 AI を用いた新ビジネスや業務改善を行いたいとしよう。おそらく上手くいくことはない。ウォーターフォールというやり方・考え方そのものが筋違いだからだ。しかし、コラボレーションモジュールを使えば、別のやり方や考え方を知ることができる。たとえばアジャイルやエクスプロラトリを知ることができる。

例2: 時間帯の合わないチームがあるとする。朝型で働く者、夜型で働く者、国をまたいでおりタイムゾーンに開きがある、など。この場合、物理的に時間が合わないためミーティングはほとんど行えない。しかし、大半のチームはミーティングの代替手段を知らないため、誰かが割りを食う羽目になる。朝型の者が夜の会議に出たり、海外本社のタイムゾーンに合わせるせいで日本拠点の社員が深夜に参加しなければならなくなったりする。このような前時代的な悲劇を減らすには、脱ミーティングをする他はない。ではそのための具体的なやり方や考え方は何か？この答えも、コラボレーションモジュールとして豊富に用意されている。たとえば非同期コミュニケーションに関するモジュールを集めてみれば、とっかかりは得やすいはずだ。

## このリポジトリの使い方（このリポジトリを使う人間向け）
利用者は Codex など AI エージェントを使って、コラボレーションに関する悩みやアイデアを尋ねよ。ChatGPT のように雑に会話を始めてもいいし、より具体的な議論に踏み込んでもいい。そうすると、このリポジトリ上のコラボレーションモジュールが提案される。現状を打開するためのやり方や考え方を知ることができるはずだ。

つまり、このリポジトリは利用者の助けになるかもしれない概念を提示する。ゼロからイチを出すことを目指している。その先は関与しないが、AI エージェントとの会話である程度の具体性や戦略を練ることはできるはずだ。

なお、Claude Code など別の AI エージェントを使っている者は、本ファイルの内容をペーストしたガイドをつくれ。たとえば Claude Code なら CLAUDE.md だろうし、Cline なら .clinerules/ 配下の Markdown ファイルとなる。

## このリポジトリの使い方（このリポジトリを使う AI エージェント向け）
AI のあなた向けの使い方を示す。

あなたの役割は、利用者からの問い合わせに応じて、最適なコラボレーションモジュールを提案することである。

コラボレーションモジュールは 1-module 1-markdown-file で書かれている。各ファイルは概念的な説明といくつかの具体例に留まっており、コストメリットや事例やストーリーといった説得用の情報は一切含めていない。説得はこのリポジトリの責務ではない。このリポジトリはあくまでも、コラボレーションモジュールを提案することで読者のゼロをイチに近づけることにフォーカスする。

もちろん、あなたは AI なので、利用者のコンテキストや問い合わせの内容に合わせて、融通を利かせれば良い。たとえば利用者の言語が英語なら英語に翻訳して答えるべきだし、利用者が厳密な定義や詳細な議論よりも端的なたとえと実例を望むのなら、たとえと例の説明を増やした方がいいだろう。

なお問い合わせに対する回答をファイルに出したい場合は、`.memo/` ディレクトリを使え。このディレクトリはバージョン管理の対象外となっている。
